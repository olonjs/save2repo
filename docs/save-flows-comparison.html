Confronto Flussi Save API

Confronto tra /api/v1/save e /api/v1/save-stream.

Flowchart

flowchart TD
  start[Client Request POST]

  start --> auth{Authorization Bearer valida?}
  auth -- No --> unauthorized[Errore Auth]
  auth -- Yes --> tenant{Tenant trovato da api_key?}
  tenant -- No --> invalidKey[Errore API Key]
  tenant -- Yes --> parseReq{path e content validi?}
  parseReq -- No --> badReq[Errore Bad Request]
  parseReq -- Yes --> githubCommit[GitHub createOrUpdateFileContents]

  githubCommit --> split{Endpoint}

  split --> savePath["/api/v1/save"]
  split --> streamPath["/api/v1/save-stream"]

  savePath --> saveResp[Response JSON success true]

  streamPath --> sseInit[SSE step/log/error]
  sseInit --> vercelChecks{Vercel config + project presenti?}
  vercelChecks -- No --> sseErrCfg[Evento error: configurazione Vercel]
  vercelChecks -- Yes --> waitDeploy[Polling deploy by commitSha]
  waitDeploy --> deployState{Deploy READY?}
  deployState -- Timeout/ERROR/CANCELED --> sseErrDeploy[Evento error: deploy failed]
  deployState -- READY --> liveUrl{URL deploy disponibile?}
  liveUrl -- No --> sseErrUrl[Evento error: URL mancante]
  liveUrl -- Yes --> sseDone[Evento done: deployUrl + commitSha]

Sequence Diagram

sequenceDiagram
  autonumber
  participant Client
  participant Api as API_SaveOrStream
  participant Supabase
  participant GitHub
  participant Vercel

  Client->>Api: POST /api/v1/save or /save-stream
  Api->>Api: Validate Bearer token
  alt Unauthorized
    Api-->>Client: Error (401 or SSE error)
  else Authorized
    Api->>Supabase: Lookup tenant by api_key
    Supabase-->>Api: Tenant record
    alt Invalid tenant or no installation
      Api-->>Client: Error (403/400 or SSE error)
    else Tenant valid
      Api->>Api: Parse path/content/message
      alt Bad request
        Api-->>Client: Error (400 or SSE error)
      else Valid payload
        Api->>GitHub: getContent(path) optional sha
        GitHub-->>Api: Existing sha or not found
        Api->>GitHub: createOrUpdateFileContents
        GitHub-->>Api: Commit created

        alt Endpoint save
          Api-->>Client: JSON success: Saved to GitHub
        else Endpoint save-stream
          Api-->>Client: SSE step commit/push done
          Api->>Vercel: Poll deployments by commitSha
          loop Until READY or timeout
            Vercel-->>Api: Deployment state
          end
          alt READY with url
            Api-->>Client: SSE done deployUrl + commitSha
          else ERROR/CANCELED/timeout/noUrl
            Api-->>Client: SSE error with code/stepId
          end
        end
      end
    end
  end
