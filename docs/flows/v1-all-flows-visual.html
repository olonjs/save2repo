JsonPages API v1 - Flussi visuali

Versione visuale dei flussi documentati in docs/flows/*.md (senza sostituirli).

1) GET /api/v1/licensing/bridge-status

flowchart TD
  A[Client subscribe] --> B[bridge-status]
  B --> C{auth ok?}
  C -- no --> C1[401/403]
  C -- yes --> D{plan/install/tenant validi?}
  D -- no --> D1[400/404/409]
  D -- yes --> E[Resolve installation]
  E --> F{installation found?}
  F -- yes --> G[state bridge_ready]
  F -- no --> H[try github_login fallback]
  H --> I{found?}
  I -- yes --> G
  I -- no --> J[state bridge_missing]
  G --> K[optional upsert billing_intents]
  J --> K
  K --> L[200 + install/configure urls]

2) POST /api/v1/licensing/create-checkout

flowchart TD
  A[Client create-checkout] --> B{auth + plan + installation valid?}
  B -- no --> B1[400/401]
  B -- yes --> C[Resolve tenant scope]
  C --> D{LS config present?}
  D -- no --> D1[500 ERR_LS_CONFIG_MISSING]
  D -- yes --> E{tenant already licensed?}
  E -- yes --> E1[409 ERR_TENANT_PLAN_ALREADY_LICENSED]
  E -- no --> F{reuse allowed? fresh+safe+context}
  F -- yes --> F1[return reused checkout]
  F -- no --> G[Call LS create checkout]
  G --> H{LS ok?}
  H -- no --> H1[persist last_error + 502]
  H -- yes --> I[upsert checkout_created]
  I --> J[200 checkoutUrl]

3) GET /api/v1/licensing/checkout-status

flowchart TD
  A[Client polling] --> B{auth + plan valid?}
  B -- no --> B1[400/401]
  B -- yes --> C[Load intent by tenant/correlation/latest]
  C --> D{intent exists?}
  D -- no --> D1[state authenticated]
  D -- yes --> E[Normalize state legacy]
  E --> F{pending checkout reusable?}
  F -- yes --> G[return checkoutUrl]
  F -- no --> H[effective state bridge_ready]
  H --> I[checkoutRecoveryRequired=true]
  G --> J[200 status]
  I --> J

4) GET /api/v1/licensing/pending-entitlements

flowchart TD
  A[Dashboard init/resume] --> B{auth ok?}
  B -- no --> B1[401/403]
  B -- yes --> C[Query licensed_ready_unassigned tenant_id null]
  C --> D[Order updated_at ASC FIFO]
  D --> E[Map + filter correlationId]
  E --> F[200 entitlements list]

5) POST /api/v1/webhooks/ls

flowchart TD
  A[LS delivery] --> B{secret+signature valid?}
  B -- no --> B1[401/500]
  B -- yes --> C[Parse payload + custom data]
  C --> D[Insert billing_webhook_events idempotency]
  D --> E{duplicate?}
  E -- yes --> E1[200 duplicate]
  E -- no --> F{userId+planCode resolved?}
  F -- no --> F1[200 ignored]
  F -- yes --> G[Load existing intent]
  G --> H[Compute next state]
  H --> I{licensed already?}
  I -- yes --> I1[prevent regression]
  I -- no --> I2[apply success/pending transition]
  I1 --> J[persist intent]
  I2 --> J
  J --> K[200 ok]

6) POST /api/v1/tenants/provision-stream

flowchart TD
  A[Dopa create now] --> B[Validate input/config]
  B --> C{source template/repository}
  C --> D[GitHub repo phase]
  D --> E[Vercel project + env + deploy]
  E --> F[Insert tenant]
  F --> G{entitlement ids provided?}
  G -- yes --> H[Atomic consume entitlement]
  H --> I{consume conflict?}
  I -- yes --> I1[SSE error ERR_ENTITLEMENT_CONSUME_CONFLICT]
  I -- no --> J[SSE done tenant+deployUrl]
  G -- no --> J

10) POST /api/v1/save

flowchart TD
  A[Editor save] --> B{Bearer api key?}
  B -- no --> B1[401]
  B -- yes --> C[Lookup tenant by api_key]
  C --> D{tenant + github_installation_id present?}
  D -- no --> D1[403/400]
  D -- yes --> E[Get file sha if exists]
  E --> F[createOrUpdateFileContents]
  F --> G[200 Saved to GitHub]

11) POST /api/v1/save-stream

flowchart TD
  A[Editor save-stream] --> B[Auth + tenant checks]
  B --> C[SSE step commit]
  C --> D[SSE step push]
  D --> E[SSE step build: trigger deploy]
  E --> F[Poll deployment by id]
  F --> G{READY?}
  G -- no --> G1[SSE error code]
  G -- yes --> H[SSE step live + persist vercel_url]
  H --> I[SSE done deployUrl+commitSha]

13) Orchestrazione: subscribe -> webhook -> entitlement -> dopa -> provision

flowchart TD
  A[User click subscribe] --> B[bridge-status]
  B --> C[create-checkout]
  C --> D[LS overlay payment]
  D --> E[webhooks/ls]
  E --> F[licensed_ready_unassigned]
  F --> G[Dopa decision]
  G -->|create now| H[provision-stream + consume entitlement]
  G -->|create later| I[pending entitlements FIFO in dashboard]
  I --> H
  H --> J[tenant visible in dashboard]
