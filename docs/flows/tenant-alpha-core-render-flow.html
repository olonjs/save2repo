Flow render: tenant-alpha -> core

Flusso utente/funzione dalla navigazione su / fino al render completo pagina. Evidenziato chi fa cosa: app tenant vs motore core. 

flowchart TD
  user["User apre '/'"] --> taMain["tenant-alpha: main.tsx avvia React"]
  taMain --> taApp["tenant-alpha: App.tsx"]
  taApp --> taData["tenant-alpha: carica pages/site/menu/theme + registry + schemas"]
  taData --> taConfig["tenant-alpha: costruisce JsonPagesConfig"]
  taConfig --> coreEngine["core: JsonPagesEngine(config)"]

  coreEngine --> coreInit["core: init theme + ConfigProvider + BrowserRouter"]
  coreInit --> routeRoot["core: route '/' -> VisitorRoute"]
  routeRoot --> slugDefault["core: slug default = home"]
  slugDefault --> bakedProbe["core: optional baked state da #jp-baked-state"]
  bakedProbe --> pageResolve["core: resolve pageConfig (baked o pageRegistry['home'])"]
  pageResolve --> pageCheck{"pageConfig esiste?"}
  pageCheck -->|no| notFound["core: DefaultNotFound / custom NotFound"]
  pageCheck -->|yes| themeLoad["core: ThemeLoader(tenant) + StudioProvider(visitor)"]
  themeLoad --> pageRenderer["core: PageRenderer"]
  pageRenderer --> sectionRender["core: SectionRenderer header/sezioni/footer"]
  sectionRender --> tenantComponents["tenant-alpha: componenti concreti dal registry"]
  tenantComponents --> finalUI["Render finale pagina"]

Responsabilita' per layer

  • tenant-alpha: prepara dati tenant, registri componenti, schemas, persistence/assets e passa config al motore.
  • core: gestisce routing, risoluzione pagina, pipeline di render e orchestrazione runtime visitor/studio.
  • Punto di contatto: <JsonPagesEngine config={...} />.
